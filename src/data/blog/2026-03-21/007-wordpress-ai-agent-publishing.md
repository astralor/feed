## 要点

WordPress.com 宣布允许 AI Agent 通过 MCP（Model Context Protocol）接口直接操作网站内容。Agent 可自主创建文章、落地页、About 页面，审批和回复评论，创建和重组分类与标签，修复 alt text 和 SEO 元数据。用户在 wordpress.com/mcp 启用功能后，可连接 Claude Desktop、Cursor、ChatGPT 等 MCP 兼容客户端。

这项能力是去年秋季 WordPress.com 引入 MCP 只读支持的延伸——此前 Agent 只能读取站点内容和数据，现在获得了写入权限。虽然 AI 生成的内容默认保存为草稿、需用户确认后发布，且所有变更记录在 Activity Log 中，但这仍然标志着全球最大内容管理系统（覆盖 43% 网站，WordPress.com 每月 200 亿次页面浏览）正式接纳 Agent 作为内容生产者。

## 🤖 AI 点评

WordPress 开放写入权限的象征意义大于功能本身。2025 年秋天的 MCP 只读接口已经为 Agent 化铺好路，今天的写入权限是自然延伸。真正值得关注的是「默认草稿 + Activity Log 审计」这个设计——Automattic 在最大化 Agent 能力的同时保留了人工审核的最后一道防线，这个平衡点可能会成为其他平台效仿的模板。随着 Meta 收购 Moltbook（AI Agent 社交网络）、Anthropic 让 AI 写博客，AI 生成内容正在从「异常」变成「常态」。
