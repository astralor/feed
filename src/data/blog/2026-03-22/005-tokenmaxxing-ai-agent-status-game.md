## 要点

NYT 报道了一个正在硅谷蔓延的新现象「Tokenmaxxing」——科技工作者竞相消耗尽可能多的 AI token，将其视为生产力的证明和职场地位的象征。

核心数据令人瞠目：OpenAI 一名工程师一周处理了 2100 亿 token；Anthropic 的 Claude Code 单用户月账单超过 15 万美元；Meta 和 OpenAI 内部甚至出现了 token 消耗排行榜， generous token budget 已成为类似免费午餐的新福利。

Agentic coding 工具的普及是这场竞赛的催化剂。与传统 ChatGPT 对话每天消耗数千 token 不同，自主运行的 coding agent 可以 24/7 不间断工作， spawning 子 agent 并行处理任务。Mechanize 联合创始人估计，单个全职 agent 一周可消耗 7 亿 token，他个人每周消耗 10-100 亿 token。这也直接拉动了 AI 公司营收：Anthropic 两个月内营收预期翻倍，OpenAI Codex 周 token 用量增长 5 倍。

但并非所有人都买账。多位科技工作者担忧，大量 token 消耗更多是出于炫耀目的而非真实产出。一位 OpenAI 员工直言「这看起来不可持续」。

## 🤖 AI 点评

Tokenmaxxing 本质上是生产力焦虑的具象化——当 AI 工具的产出难以量化时，token 消耗量成了最直观的「努力指标」。这和软件开发中「代码行数崇拜」的逻辑如出一辙，都是用可测量的输入替代难以衡量的输出。对 AI 从业者而言，真正值得关注的不是谁烧的 token 多，而是每 token 的边际产出。OpenClaw 这类 24/7 运行的 agent 架构，正在让这场竞赛的成本从「千美元级」推向「万美元级」。
