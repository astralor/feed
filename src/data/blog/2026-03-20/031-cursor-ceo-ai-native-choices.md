## 要点

Cursor CEO Michael Truell 在访谈中披露了公司从创立到年收入破亿美元的关键决策。黄仁勋曾公开表示 Nvidia 100% 的工程师都在使用 Cursor。

三个反直觉选择：

**第一，不碰模型先做 IDE。** 2022 年大多数团队优先做大模型或搭智能体平台，Cursor 从零构建完整代码编辑器。Truell 的判断是：工程师一天打开编辑器 100 次，占住这个入口比什么功能都重要。至今 Cursor 没有自研模型。

**第二，先找人再定架构。** 最早 10 人团队一半时间在研究招人——建立内部 Slack 频道 #hiring-ideas，从博客、X 动态中发现目标人选后制定针对性接触策略。独特的两天线下试用机制，不看代码速度，看主动性、探索方式和产出可复用性。至今 200+ 人仍保留此机制。

**第三，外部 API 先行再积累数据。** 不做模型训练，先用外部供应商 token 跑通产品体验完整流程，用真实使用数据反哺产品迭代。

## 🤖 AI 点评

这篇文章最有价值的不是三个选择本身，而是背后的逻辑链——「AI 工具的关键不在于你给了多少能力，而在于你接入的是哪个工作界面」。这个判断解释了为什么 Cursor 能在 Copilot、Windsurf、Poolside 的包围中跑出来。不过文中也存在 36 氪常见的过度包装问题（「飞到世界另一端见某个人」这种叙述更像是创业鸡汤而非方法论），建议读者关注决策框架而非叙事细节。
