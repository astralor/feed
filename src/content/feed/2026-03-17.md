## 🌟 多 Agent 系统突破

### Spine Swarm – 可视化画布上的 AI agents 协作
**来源**: Hacker News | **评分**: 9/10

YC S23 项目 Spine Swarm 推出多 agent 系统，在无限可视化画布上完成复杂非编码项目（竞品分析、财务建模、SEO 审计、pitch deck、交互原型等）。

**核心创新**：
- **画布架构**：Agent 通过画布存储中间结果，避免传统多 agent 系统的上下文退化问题
- **基准表现**：在 Google DeepMind DeepSearchQA（900 个跨 17 领域的因果链问题）上得分 87.6%，零人工干预
- **可审计性**：每步都完全可审计，甚至发现了 GAIA Level 3 基准中的错误答案

**架构特点**：
- 中央编排器将任务分解为子任务，委托给专业 persona agent
- Agent 可覆盖默认设置（主要是模型和提示），为每个子任务选择最佳模型
- 多个 agent 在子任务无依赖时并行工作，下游 agent 自动接收上游工作的上下文

[了解更多](https://www.getspine.ai/)

## 🔧 Agent 基础设施工具

### Context Gateway – 压缩 agent 上下文
**来源**: Hacker News | **评分**: 8/10

开源代理，位于编码 agent（Claude Code、OpenClaw 等）和 LLM 之间，在进入上下文窗口前压缩工具输出。

**问题背景**：
- Agent 在管理上下文方面很糟糕
- 单次文件读取或 grep 可能向窗口倾倒数千个 token，大部分是噪音
- 长上下文基准显示准确率随上下文增长急剧下降（OpenAI GPT-5.4 从 32k 的 97.2% 降至 1M 的 36.6%）

**解决方案**：
- 使用小型语言模型（SLM）查看模型内部，训练分类器检测上下文中信号最强的部分
- 根据工具调用意图进行压缩（如 agent 调用 grep 查找错误处理模式，SLM 保留相关匹配并剥离其余部分）
- 如果模型后续需要被移除的内容，可调用 expand() 获取原始输出
- 在 85% 窗口容量时进行后台压缩，延迟加载工具描述

[GitHub 仓库](https://github.com/Compresr-ai/Context-Gateway)

### Axe – 12MB 的 AI Agent 框架
**来源**: Hacker News | **评分**: 8/10

Go 编写的 12MB 二进制 AI agent 框架，将 LLM agent 视为 Unix 程序。

**核心理念**：
- 每个 agent 是一个 TOML 配置，专注于单一任务（代码审查、日志分析、commit 消息编写）
- 可从 CLI 运行，管道输入数据，获取结果输出
- 可用管道链接，或从 cron、git hooks、CI 触发

**特性**：
- 12MB 二进制，两个依赖，无框架、无 Python、无 Docker
- Stdin 管道：`git diff | axe run reviewer` 直接工作
- Sub-agent 委托：Agent 通过工具使用调用其他 agent，深度限制
- 持久化内存：Agent 可跨运行记住，无需管理状态
- MCP 支持：可连接任何 MCP 服务器
- 内置工具：web_search 和 url_fetch 开箱即用
- 多提供商：Anthropic、OpenAI、Ollama 或任何 models.dev 格式
- 路径沙盒文件操作：将 agent 锁定到工作目录

[GitHub 仓库](https://github.com/jrswab/axe)

### Agent-browser-protocol – AI Agent 专用浏览器
**来源**: Hacker News | **评分**: 8/10

Fork Chromium 构建的 agent-browser-protocol (ABP)，解决浏览器-agent 交互中的状态同步问题。

**核心问题**：
- 大多数浏览器-agent 失败不是因为模型误解页面，而是模型从过时状态推理
- 模态框在最后一次 Playwright 截图后出现并阻塞输入
- 动态过滤器导致页面在步骤之间重排
- 自动完成下拉框打开并覆盖 agent 打算点击的元素

**解决方案**：
- 在每次操作（点击、输入等）后冻结 JavaScript 执行和渲染，捕获结果状态
- 编译该操作循环期间发生的显著事件（导航、文件选择器、权限提示、警报、下载）
- 将冻结的页面状态截图和结构化摘要一起发送回 agent

**性能**：
- 在 Online Mind2Web 基准上用 Opus 4.6 得分 90.5%

[GitHub 仓库](https://github.com/theredsix/agent-browser-protocol)
